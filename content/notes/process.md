>[!note] Process
>A process is an instance of an executing program. It has some state in memory such as:
>- The code that is being run,
>- Initial data it started with,
>- The [heap](heap_(os).md) associated with the application, and
>- The [stack](stack_(os).md) associated with the application.
>This all lives in the processes [address space](address_space_(os).md).
